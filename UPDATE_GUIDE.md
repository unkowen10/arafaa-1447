# 📤 طريقة تحديث الموقع على GitHub Pages بدون تغيير اللينك

اللينك بتاع GitHub Pages بيكون شكله:
```
https://USERNAME.github.io/REPO/
```
ولا يتغير أبداً طول ما الـ Repository اسمه ثابت. اللي بيتغير هو **محتوى الملفات** بس.

---

## ✅ خطوات التحديث (نفس اللينك يفضل شغال)

### الطريقة 1: من الموقع مباشرة (الأسهل)
1. ادخل على الـ Repository بتاعك على GitHub.
2. اضغط على الملف اللي عاوز تعدله (مثلاً `duas.html`).
3. اضغط أيقونة **القلم ✏️** (Edit this file).
4. عدّل اللي انت عاوزه.
5. تحت تلاقي **Commit changes** → اكتب وصف مختصر → اضغط **Commit changes**.
6. ✅ خلاص! GitHub Pages هينشر التحديث خلال 30 ثانية إلى دقيقتين.

### الطريقة 2: ترفع الملفات الجديدة بدل القديمة
1. ادخل الـ Repository.
2. اسحب الملفات الجديدة (مثل `duas.html`, `style.css`, `app.js`, `sw.js`) من جهازك واسقطها داخل صفحة الـ Repo.
3. اعمل Commit.

### الطريقة 3: من Git (لو شغال محلياً)
```bash
git add .
git commit -m "تحديث الأدعية وإصلاح مشكلة القص"
git push
```

---

## ⚠️ مهم جداً عند كل تحديث (علشان الناس تشوف التحديث)

الموقع PWA وبيستخدم **Service Worker** بيخزن الملفات في كاش المتصفح. لو ما عدلتش رقم النسخة، الناس مش هتشوف التحديث.

### الحل (في كل مرة بتحدث):
1. افتح ملف `sw.js`.
2. غير السطر:
   ```js
   const CACHE_VERSION = 'v5-2026-05-25';
   ```
   إلى رقم/تاريخ جديد مثلاً:
   ```js
   const CACHE_VERSION = 'v6-2026-05-26';
   ```
3. ارفع التعديل.
4. أول ما حد يفتح الموقع: هيظهرله **toast** في الأسفل مكتوب فيه "🔄 يوجد تحديث جديد للموقع" → بيضغط "تحديث الآن" → تمام.

> الـ HTML بقى يستخدم استراتيجية **Network-first** يعني هيحاول يجيب آخر نسخة من السيرفر الأول، فالتحديثات في النصوص هتظهر فوراً حتى من غير ما تغير الـ CACHE_VERSION، بس **الـ CSS و JS** محتاجين تغيير النسخة علشان يتجدد الكاش بتاعهم.

---

## 🛠️ لو حد ما شافش التحديث (حل المستخدم)
- خروج من الموقع وفتحه تاني.
- لو لسه: Ctrl+Shift+R أو من إعدادات المتصفح → Clear site data.
- في الموبايل: حذف التطبيق المثبت وإعادة التثبيت.

---

## 📂 تنظيم الملفات على GitHub
لو الملفات في root الـ Repository، تأكد إن مسارات `sw.js` صح:
- لو حاطط CSS داخل مجلد `css/`، خليها كده.
- لو في الـ root مباشرة، عدل `sw.js`:
  ```js
  'style.css', 'app.js'
  ```
  بدلاً من `'css/style.css', 'js/app.js'`.

---

## 🧪 اختبار محلي قبل الرفع
افتح Terminal في فولدر الموقع وشغل:
```bash
python3 -m http.server 8000
```
ثم افتح: http://localhost:8000
